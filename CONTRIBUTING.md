# Contributing to OFFGRID

Thanks for helping improve OFFGRID.

## Before you open a PR

- Keep changes small and focused.
- Preserve the browser-only model unless there is a strong reason not to.
- Test the app in a real browser, ideally from `http://localhost` and `file://`.
- Make sure any added UI still works on mobile widths.

## Good PRs

- Fix a real bug or UX problem.
- Include before/after screenshots for visual changes.
- Update the README if the user flow changes.

## Code style

- Prefer vanilla browser APIs.
- Keep the single-file structure unless you are intentionally splitting it.
- Avoid adding dependencies unless they solve a clear problem.
